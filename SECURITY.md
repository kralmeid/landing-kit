# Security Policy

## Supported Versions

`landing-kit` is a starter template. Security fixes target the default branch:

| Version | Supported |
| --- | --- |
| `main` | Yes |

Tagged releases may be created for stable snapshots, but the default branch is the source of truth for active security updates.

## Reporting a Vulnerability

Do not open a public issue for suspected vulnerabilities.

Report security issues through GitHub private vulnerability reporting if available for this repository, or contact the maintainer directly through the email listed on the maintainer's GitHub profile.

Include:

- Affected file, package, workflow, or configuration.
- Reproduction steps or proof of impact.
- Whether the issue affects generated projects, this starter repository, or both.

Expected response: the maintainer will review valid reports, patch supported code when appropriate, and publish a release note or changelog entry when disclosure is safe.
